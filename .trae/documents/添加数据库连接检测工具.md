## 目标
在后台管理页面增加一个数据库连接检测工具，用于分析数据库连不上的原因。

## 实现方案

### 1. 创建 API 端点
新建文件 `src/app/api/admin/db-check/route.ts`
- 检测环境变量配置状态（DATABASE_URL、SUPABASE_URL、SUPABASE_ANON_KEY）
- 尝试连接数据库并返回详细错误信息
- 检测网络连通性
- 返回结构化的诊断报告

### 2. 创建前端组件
新建文件 `src/components/admin/DbConnectionChecker.tsx`
- 显示连接状态卡片（环境变量、数据库连接、网络状态）
- 提供"重新检测"按钮
- 显示详细的错误信息和解决建议
- 使用 UI 组件库中的 Card、Button、Badge 等组件

### 3. 集成到管理后台
修改 `src/app/admin/dashboard/page.tsx`
- 在 Tabs 中新增"数据库检测"标签页
- 引入 DbConnectionChecker 组件

### 4. 检测内容
- **环境变量检查**: 检查 DATABASE_URL、SUPABASE_URL、SUPABASE_ANON_KEY 是否配置
- **Prisma 连接测试**: 尝试使用 PrismaClient 连接数据库
- **Supabase 连接测试**: 尝试使用 Supabase 客户端连接（开发环境）
- **网络连通性**: 检测基本网络状态
- **错误诊断**: 根据错误类型提供具体解决建议

## 文件变更
